# 分支管理
1. 一般来说每开发一个组件或者开发到一个阶段就要在一个新的分支上进行操作,开发完成后就合并到master分支上
2. 开发功能的时候,会增加一个分支,测试没有问题之后再合并到主分支上
3. 在分支上的修改不会影响到主分支
4. 主分支可以把分支上的代码合并下来 (把代码修改成分支上的代码),
   同理, 分支也可以把主分支上的代码合并下来 (把代码修改成主分支上的代码)

## 步骤
1. 在线上创建分支

2. 在本地运行命令把线上的分支拉取下来
``` bash
git pull
```

3. 切换分支(这样本地的操作就会都在这个分支上了)
``` bash
git checkout 分支名
```

* 查看本地分支
``` bash
git status
```

4. 合并分支
``` bash
# 先推送到当前分支上
git add .
git commit -m '推送到分支'
git push
# 切换到master分支上
git checkout master
# 合并到分支
git merge origin/分支名
# 推送到master分支
git push
```
